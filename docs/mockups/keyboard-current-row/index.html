Focus versus current

The focus ring marks the active keyboard target. The current-row cue marks the app's current item. They can appear on different rows or on the same row.

Expanded and collapsed rows

The mockup keeps expansion icons and state text visible so current or focus styling does not hide the hierarchy cue.

Interactive controls

Buttons marked with data-tree-view-interactive represent host-app-owned actions. The mockup does not define routes, authorization, or final action copy.

Review boundaries

  • Use this page to compare focus-visible and current-row styling when both cues are present in the same tree.
  • Use keyboard-focus-states.html when review only needs representative focus targets without current-row context.
  • Keep tab order, arrow-key behavior, roving tabindex, routes, permissions, and business actions outside this static mockup.
