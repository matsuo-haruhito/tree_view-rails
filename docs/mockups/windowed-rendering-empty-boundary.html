Beyond-offset state

Requested offset past visible rows

requested offset: 160 limit: 12 total_count: 143

TreeView boundary: no rows are rendered because the requested offset is outside the visible-row list. Metadata still gives the host app enough information to return to the previous available window.

Metadata Representative value Review meaning
rows [] The window is empty; TreeView does not invent fallback rows.
before_count 143 The count is capped at total_count, not the requested offset.
after_count 0 No visible rows remain after a beyond-offset request.
previous_offset 131 The host app may use this as a return target for the last populated window.
next? false The host app should not present a forward action as TreeView-owned behavior.
Rendered row area is intentionally empty for this representative state.

Responsibility boundary

Use this page to review metadata copy and empty-window affordances without designing host-app pagination.

  • TreeView owns visible-row slicing, empty rows, capped counts, and previous / next metadata.
  • Host app owns offset persistence, route params, disabled button treatment, fallback copy, and whether to redirect to the previous populated window.
  • Not modeled here server-side pagination helpers, infinite scroll, cursor encoding, authorization, and screenshot baselines.
