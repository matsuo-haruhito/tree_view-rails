What this mock compares

Surface Best for Keep outside scope
Plain draggable row Confirming the baseline row stays draggable when no control marker is present. Drop targets, persistence, authorization, and route wiring.
Native interactive controls Checking that built-in controls remain safe to click or edit without extra TreeView markers. Host-app validation, save timing, or business copy.
data-tree-view-interactive Showing the broad custom-widget marker for non-native controls inside a draggable row. Widget implementation details or unrelated keyboard and row-click policy changes.
data-tree-view-ignore-drag Showing the narrower marker when drag start alone should be ignored. Business-specific drag affordances, recovery UI, or drop sequencing rules.
  • These markers are shown as reusable TreeView guard hooks, not as a full drag-and-drop workflow.
  • Final labels, permissions, destination rules, and save behavior remain host-app responsibilities.
