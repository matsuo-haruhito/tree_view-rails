Row-level actions are shown as host-owned controls inside a TreeView row. This mockup keeps the menu visual state reviewable while leaving routing, authorization, keyboard command policy, and menu controller behavior outside TreeView.

Review intent

  • Action controls do not become a TreeView-owned routing or authorization surface.
  • The menu trigger and items are explicitly marked interactive so row selection, drag, and toggle gestures can ignore them.
  • Open state is visual only; runtime menu behavior stays with the host app.

Boundary checklist

row_actions_partial supplies the trigger/items markup.
data-tree-view-interactive marks controls that should not toggle or select the row.
data-tree-view-ignore-drag keeps drag handles from capturing menu gestures.
Native button/link semantics remain visible for accessibility review.
