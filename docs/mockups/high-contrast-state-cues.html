Forced-colors framing

The sample includes a small forced-colors rule so borders and outlines remain inspectable, but it is still a static review surface rather than a complete theme system.

State ownership

TreeView can expose durable row hooks and baseline cues. The host app chooses final colors, status language, route policy, and business actions.

Related pages

Use keyboard focus and interaction-state mockups for deeper inspection of focus targets, loading, retry, and drag/drop behavior boundaries.

Review checklist

  • Current, selected, focused, error, and drop-target rows remain distinguishable without relying only on fill color.
  • State text is visible in the table, so screenshots can be reviewed without knowing the color palette.
  • The page avoids changing runtime JavaScript, ARIA semantics, public API, route policy, or host-app authorization behavior.
