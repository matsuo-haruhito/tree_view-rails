Table-first, not treegrid

The sample stays in a normal table and does not claim role="treegrid", roving tabindex, or a complete WAI-ARIA tree keyboard model.

ARIA placement

Representative row-level state is visible beside the toggle-link state so reviewers can check what belongs to rows versus controls.

Review checklist

  • Row-level aria-level, aria-expanded, aria-selected, and aria-current samples are visible without implying a full treegrid model.
  • Toggle links expose aria-expanded while aria-controls, caption policy, keyboard shortcuts, and focus order remain host-app decisions.
  • Runtime Ruby, JavaScript, helper output, role mapping, and automated accessibility tooling are intentionally unchanged.
