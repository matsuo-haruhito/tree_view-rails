Static HTML for reviewing how a table-first tree can stay legible in a narrow sidebar or small viewport. The goal is not to ship a responsive design system. The mock keeps hierarchy cues, the primary label, and current or selection state visible while less-critical metadata wraps below.

Review cues

  • Do not hide the toggle path or the primary label before secondary metadata has somewhere else to go.
  • Current, selected, loading, and archived state should remain visible without forcing a wide multi-column row.
  • Exact truncation, compact menus, and breakpoints remain host-app responsibilities; this mock is a product-neutral reference.
