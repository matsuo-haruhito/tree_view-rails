Focused Turbo Frame boundary

Turbo Frame target mockup

Static reference for the boundary between the TreeView-owned toggle link attribute and the host-app-owned Turbo Frame target, controller response, and authorization checks.

Target frame and link attribute

The frame placement is supplied by the host app. TreeView only adds the configured data-turbo-frame value to expand or collapse links.

Host app frame target <turbo-frame id="documents_tree">
Document Status Owner
Collapse Documents Current data-turbo-frame="documents_tree" Open Records team
Expand Policies data-turbo-frame="documents_tree" Collapsed Compliance
Leaf Quarterly review.pdf Read only Finance

TreeView owns

The configured turbo_frame: "documents_tree" value is rendered on toggle links as data-turbo-frame.

Host app owns

The surrounding <turbo-frame>, route, controller action, authorization, and Turbo Stream or HTML response stay outside the gem.

Review focus

Use this page to confirm that reviewers can see which links target the frame without implying that TreeView supplies navigation behavior.

This mockup is static. It does not perform Turbo navigation, fetch children, or model host-app permission copy.
