Children pagination selection boundary mock

Focused static reference for reviewing checkbox selection when a branch shows only a loaded slice of children. The examples keep TreeView's rendered-row boundary visible and leave server-side selection intent, authorization, and final bulk actions with the host app.

Rendered-row cascade and indeterminate cues

The parent can show an indeterminate review cue when only some rendered descendants are selected. The static mockup avoids promising cascade behavior for rows that have not been loaded.

Rendered-only rule: the visual mixed state explains the loaded slice; it does not choose a product policy for unloaded descendants.

Host-app bulk action boundary

When a bulk action should include descendants that are not rendered, the host app needs a query-backed or server-side intent. TreeView can keep the visible-row payload honest without owning that workflow.

DOM-submitted selection

Best for actions on loaded checked rows.

tree_selection[] = { id: "planning", depth: 1 }
tree_selection[] = { id: "delivery", depth: 1 }

Query-backed intent

Best when the host app offers "all descendants" across unloaded pages.

selection_intent = { branch: "roadmap", scope: "all_descendants" }

Review boundary

Keep final copy, confirmation, permission checks, and server validation outside this static TreeView mockup.
