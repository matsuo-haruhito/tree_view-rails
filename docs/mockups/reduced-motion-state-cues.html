tree_view-rails

TreeView reduced-motion state cue mock

Static reference for reviewing loading, retry, selected/current, and drop-target cues when motion is reduced or unavailable. The examples use text, row shape, borders, and adjacent status labels instead of animation-only feedback.

Back to review gallery Open mockup index

Responsibility boundary

TreeView-owned cues

Row state classes, depth structure, toggle cell layout, and ARIA state remain visible without requiring animation.

Host-app-owned cues

Final copy, retry wording, persistence status, and product-specific confirmation messages stay with the host app.

Out of scope here

No animation system, JavaScript timing, Turbo response, or visual regression baseline is introduced by this mockup.
