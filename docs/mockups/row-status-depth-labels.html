Row status and depth labels

Focused static reference for comparing row-wide status cues, disabled selection checkboxes, and depth labels without turning the mockup into a host-app authorization or business-copy example.

Row status stays row-wide

Compare row-wide disabled and readonly cues against a disabled checkbox state. The status badge colors the row context; the checkbox-disabled reason only explains why selection is blocked for that row.

select tree name status cue review note
hide root folder L0 Operations workspace active Normal row without selection restrictions.
child folder L1 Quarter close archive readonly The row cue explains the broader state even though selection still uses the ordinary checkbox path.
child folder L1 Security hold queue disabled The blocked checkbox is narrower than the row cue. It only explains selection, not every business action on the row.

Depth labels stay structural

Depth labels help reviewers see hierarchy position at a glance. They do not claim business meaning on their own, so the mock keeps them short and product-neutral.

tree label cue what stays outside TreeView
root folder L0 Whether the root means portfolio, region, or category stays host-app owned.
child folder L1 Styling and wording can change in the host app without changing the reusable hierarchy output.
leaf file L2 Depth cue remains separate from approval, archive, or permission states.
