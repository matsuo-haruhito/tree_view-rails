Same node key, different browser targets

Node keys identify tree data for expansion, selection, persisted state, and diagnostics. DOM IDs identify rendered HTML targets and can change with the tree instance prefix.

Review boundary

  • Tree node key: stable data identity.
  • DOM ID: browser target scoped by UI config and tree instance.
  • Final business labels, permissions, and route names stay host-app owned.

What to review

  • Node keys remain stable across tree instances.
  • DOM IDs remain unique for browser targets.
  • Type badges and localized labels do not redefine identity.

Out of scope

  • Key generation API redesign.
  • GraphAdapter traversal or persistence behavior.
  • Host-app route, authorization, or final copy decisions.
