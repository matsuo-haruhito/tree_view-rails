tree_view-rails

TreeView mockup review gallery

Static review hub for comparing the main TreeView mockups and the focused toolbar or resource-table references without running a Rails app. Each card keeps the existing mockup as the source of truth and adds just enough framing to compare baseline structure, narrow layouts, interaction states, empty-state spacing, and newer focused surfaces in one session.

How to use this gallery

  • Start here when a review needs quick side-by-side comparison across the main TreeView mockup surfaces.
  • Use the focused toolbar and resource-table cards when one review thread is about tree-wide actions or table-owned column context rather than the whole mockup family.
  • Open the linked full mockup when you want the complete page context or longer notes for one state family.
  • Treat final business copy, routes, permissions, and actions as host-app responsibilities. This gallery stays product-neutral on purpose.

Comparison cues

Surface Best for Keep outside scope
Default tree Baseline DOM shape, hierarchy path, selection cues, badges, and row actions. Host-app CRUD, queries, business labels, or authorization behavior.
Narrow sidebar Hierarchy readability when width shrinks and metadata needs to wrap. Exact breakpoints, truncation rules, or a shipped responsive design system.
Interaction states Loading, retry, pagination, and drag or drop status cues. Real Turbo responses, route wiring, or persistence behavior.
Empty state Spacing, wrapper hooks, and the full-width empty-row slot. Final onboarding copy, CTA wording, permission messaging, or filter reset logic.
Toolbar actions Tree-wide expand and collapse states, including which action stays available in fully expanded or fully collapsed trees. Route names, authorization, or business-specific toolbar copy.
Resource table bridge Shared hierarchy-row readability when a separate table layer owns visible columns and the layout changes between fuller and compact column sets. Table preference controls, host-app query strategy, or runtime bridge wiring.
