Bridge responsibilities at a glance

Use this mock when review needs a visual reference for table-owned columns plus TreeView-owned hierarchy cues, without pulling in controllers, persistence, or host-app business actions.

Table layer owns columns

Column order, visibility, labels, and saved table state stay outside TreeView.

TreeView owns hierarchy rows

Indent guides, toggle controls, row depth, and parent or child shape stay consistent across column sets.

Host app owns final actions

Queries, routes, permissions, badges with business meaning, and workflow-specific labels remain app concerns.

Review checklist

Review concern What this mock should answer Keep outside scope
Column sharing The same tree rows stay readable across a fuller, narrower, and near-minimal visible column set. Automatic column inference, saved preference UI, or table personalization controls.
Hierarchy cues Indent guides, toggles, current-row emphasis, and hidden-child counts remain legible in the first data column. Real expand or collapse behavior, controller wiring, or asynchronous loading.
Minimal column recovery The near-empty table state still shows the primary label, current cue, and host-owned row boundary without pretending TreeView owns column recovery. Saved table preferences, required action policy, authorization-aware columns, or column recovery UI.
Responsibility boundary TreeView owns the hierarchy presentation while the table layer owns which non-tree columns appear. Host-app query strategy, authorization, routing, and business-specific row actions.
