State cues to review

VariantWhat it showsWhat stays out of scope
Enabled setAll three actions remain available while the tree contains a mix of expanded, collapsed, and current-path branches.Route names, Turbo wiring, or action sequencing rules.
Expand disabledThe tree is already fully expanded, so metadata can render expand-all as a disabled fallback while collapse actions remain available.Business copy about why a branch is open or who may collapse it.
Collapse disabled or missing pathThe tree is already fully collapsed, so collapse actions can render as disabled fallbacks, including the case where current-path metadata has no usable path.Authorization, route generation, saved preferences, or persistence across visits.
Current path preservedThe current-path action reads as the active state while ancestors stay open and sibling branches collapse away.The semantics of choosing the current path or how the host app stores that state.
Long or localized labelsA compact toolbar can wrap longer labels while preserving icon alignment, disabled styling, and current-state emphasis.Final translations, product-specific action names, exact truncation rules, or host-app breakpoints.
Metadata boundaryAction availability and fallback state stay visible without making this mockup a contract for tree_view_toolbar_action_metadata return shape.Manifest-backed public API decisions, helper return-shape guarantees, or host-app permission logic.
