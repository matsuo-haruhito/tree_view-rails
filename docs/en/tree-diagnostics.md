# Tree diagnostics

This page explains diagnostics helpers for inspecting TreeView structure and investigating data issues.

## Overview

Tree diagnostics help inspect tree structure, data consistency, and rendering risk.

They are useful for checking:

- node key uniqueness
- DOM ID collisions
- orphan records
- parent paths
- tree stats
- cycle diagnostics
- expanded keys for paths

Use them during development, before release, or when investigating inconsistent host-app data.

## Node key uniqueness

Enable validation when building the tree to catch duplicate node keys early.

```ruby
tree = TreeView::Tree.new(
  records: documents,
  parent_id_method: :parent_document_id,
  validate_node_keys: true
)
```

Duplicate keys raise `ArgumentError`.

## DOM ID collisions

Use `RenderState#validate_unique_dom_ids!` to check generated DOM IDs.

```ruby
render_state.validate_unique_dom_ids!
```

The validation includes DOM IDs generated by configured features such as selection checkboxes.

## Orphan diagnostics

In records mode, use orphan helpers to find records whose parent is missing from the record set.

```ruby
tree.orphan_items
```

`orphan_strategy` controls how orphans are handled: ignored, shown as roots, shown as orphans only, or raised as errors.

```ruby
tree = TreeView::Tree.new(
  records: documents,
  parent_id_method: :parent_document_id,
  orphan_strategy: :as_root
)
```

## Parent path helpers

Records mode can inspect parent paths.

```ruby
tree.parent_for(document)
tree.ancestors_for(document)
tree.path_for(document)
tree.paths_for(documents)
```

Use path helpers to build expanded keys for search results or current nodes.

```ruby
expanded_keys = tree.expanded_keys_for_paths([document])
```

## Tree stats

Use stats helpers to inspect tree size and reachable nodes.

```ruby
tree.stats
```

These signals can help decide whether to use render scope, windowed rendering, or lazy loading for large trees.

## Cycle diagnostics

Cycles in parent relationships can break descendant counts or path traversal.

Cycle diagnostics help investigate invalid parent-child data during development.

```ruby
TreeView::CycleDiagnostics.new(tree).report
```

## When to use diagnostics

- Check DOM ID and node key collisions before release.
- Inspect orphans after imports or migrations.
- Build expanded keys for initial expansion to search results.
- Decide rendering strategy for large trees.
- Investigate invalid parent relationships in host-app data.

## Responsibility boundary

| Area | TreeView | Host app |
|---|---|---|
| diagnostics helper | yes | calls helper |
| duplicate key detection | yes | chooses stable keys |
| DOM ID collision detection | yes | fixes config/prefixes |
| orphan detection | yes | decides data policy |
| cycle report | yes | fixes data |
| rendering strategy decision | provides signals | chooses behavior |
