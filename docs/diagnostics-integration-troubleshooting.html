CSS and JavaScript integration states

Ready

Styles and controller loaded

Tree rows have spacing, disclosure affordances, and keyboard focus treatment. Expand and collapse actions remain discoverable.

CSS missing

Unstyled tree

Markup is present, but spacing and status color are missing. The message should direct reviewers to asset loading, not data cleanup.

JavaScript missing

Static disclosure controls

Buttons render but do not expand or collapse nodes. The copy should name the controller registration and remain framework-neutral.

Route gap

Turbo response unavailable

Lazy loading cannot fetch children. The message should keep the boundary at route wiring and avoid implying permission changes.

Review checklist

  1. Confirm diagnostic states are visually stronger than integration warnings because they can block safe rendering.
  2. Confirm each integration state keeps responsibility with asset, controller, or route wiring instead of data shape.
  3. Confirm responsive layout stacks the state cards without hiding status labels or row metadata.
  4. Confirm the reference remains documentation-only and does not change generated component behavior.
